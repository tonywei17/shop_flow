# 見積もり変更に関する詳細説明資料 / 报价变更详细说明文档

## 1. 概要 / 概述

本資料は、eラーニング機能の実装方針を「自社開発（プランA/B）」から「他社eラーニングシステムとの連携（プランC）」に変更した場合の見積もり構成の変化、および技術的な考慮事項をまとめたものです。

本资料旨在总结e-learning功能实施方针从“自主开发（方案A/B）”变更为“与第三方e-learning系统对接（方案C）”时，报价构成的变化及相关的技术考量事项。

---

## 2. 見積もり構成の変更点 / 报价构成的变更点

### 2.1 削減される項目 / 削减的项目
他社システムを利用することで、以下のUI/UX開発工数が削減されます。
使用第三方系统可以削减以下UI/UX开发工时。

| 項目 (项目) | 内容 (内容) | 理由 (理由) |
| :--- | :--- | :--- |
| **動画再生機能**<br>(视频播放功能) | プレイヤー制御、再生位置保存、倍速再生など<br>(播放器控制、播放进度保存、倍速播放等) | 他社システム側の機能を利用するため<br>(直接使用第三方系统功能) |
| **学習画面UI実装**<br>(学习界面UI实现) | カリキュラム一覧、進捗バー、完了判定UI<br>(课程大纲、进度条、完成判定UI) | 他社システムの画面を表示するため<br>(直接显示第三方系统界面) |
| **動画管理CMS**<br>(视频管理CMS) | 動画アップロード、公開設定画面<br>(视频上传、发布设置界面) | 他社システムの管理画面を利用するため<br>(使用第三方系统的后台管理) |

### 2.2 追加・増額される項目 / 新增・增加的项目
システム間の連携（API・認証）に関わる工数が新たに追加されます。
新增了涉及系统间对接（API、认证）的相关工时。

| 項目 (项目) | 内容 (内容) | 重要度 (重要度) |
| :--- | :--- | :--- |
| **SSO認証基盤構築**<br>(SSO认证基建搭建) | ユーザーが再ログインなしで他社システムへ遷移するための仕組み (SAML/OIDC)<br>(实现用户无需重新登录即可跳转至第三方系统的机制) | **必須 (必须)**<br>UX担保のため |
| **会員データ連携API**<br>(会员数据同步API) | 会員情報の作成・更新・削除をリアルタイムで同期する機能<br>(实时同步会员信息的创建、更新、删除功能) | **必須 (必须)**<br>データ不整合防止 |
| **学習履歴受信API**<br>(学习记录接收API) | 「誰が・いつ・どの動画を見たか」というデータを他社から受け取る機能 (Webhook等)<br>(接收“谁・何时・看了哪个视频”的数据功能) | **必須 (必须)**<br>資格認定のため |
| **外部仕様調査・調整費**<br>(外部规格调研与调整费) | 他社システムのAPI仕様調査、技術担当者との定例MTG、問い合わせ対応<br>(第三方API规格调研、与技术负责人的例会、咨询应对) | **高 (高)**<br>開発リスク管理 |
| **結合テスト実施費**<br>(集成测试实施费) | 自社環境と他社環境を実際に接続しての動作検証<br>(实际连接双方环境进行动作验证) | **必須 (必须)** |

## 3. API連携パターンと費用感 / API对接模式与费用预估

連携の主導権がどちらにあるかによって、開発難易度と費用が変動します。
根据对接的主导权在谁手中，开发难度和费用会有所波动。

### パターン1：御社主導（推奨） / 模式1：贵司主导（推荐）
**「御社がデータ形式を決め、他社システムがそれに合わせる」**
**“由贵司定义数据格式，第三方系统配合适配”**

*   **メリット (优点)**: 
    *   自社システムの改修が最小限で済む (本公司系统改动最小)。
    *   将来的なシステムの切り替えが容易 (未来更换系统容易)。
*   **デメリット (缺点)**: 
    *   他社システム側のカスタマイズ費用が高額になる可能性がある (第三方系统的定制费用可能较高)。
*   **開発工数**: 中 (开发工时：中)

### パターン2：他社システム主導 / 模式2：第三方系统主导
**「他社システムの仕様に、御社が合わせて開発する」**
**“贵司配合第三方系统的规格进行开发”**

*   **メリット (优点)**: 
    *   一般的なSaaS等を利用する場合、導入がスムーズ (使用通用SaaS时，导入顺畅)。
*   **デメリット (缺点)**: 
    *   相手の仕様変更の影響を直接受ける (直接受对方规格变更影响)。
    *   データ変換処理（Middleware）の開発が必要 (需要开发数据转换中间件)。
*   **開発工数**: 高 (开发工时：高 - 需包含调研与转换成本)

## 4. 総合的なコスト比較 / 综合成本比较

| プラン (方案) | イニシャル開発費 (初期开发费) | ランニングコスト (运维成本) | リスク (风险) |
| :--- | :--- | :--- | :--- |
| **A/B (自社開発)** | **標準** (Standard) | 低 (サーバー費のみ)<br>低 (仅服务器费) | 動画配信トラフィック<br>视频流量 |
| **C (他社連携)** | **標準〜高** (Standard ~ High)<br>※連携難易度による | **高** (他社システム利用料)<br>高 (第三方系统使用费) | 連携トラブル、仕様変更<br>对接故障、规格变更 |

**結論**: 
UI開発費は下がりますが、**「認証・API連携・調整費」がそれ以上に発生する可能性**があり、トータルコストが安くなるとは限りません。
**结论**: 
虽然UI开发费降低了，但**“认证、API对接、沟通调整费”可能会产生更高的费用**，总成本未必会降低。

## 5. 顧客への説明ガイド / 给客户的说明指南

### 日本語説明スクリプト (日语说明脚本)

**導入 (开场):**
「ご相談いただいた他社eラーニングシステムとの連携について、技術的な観点から見積もり構成を整理いたしました。」
(针对您咨询的与第三方e-learning系统对接一事，我们从技术角度整理了报价构成。)

**コスト構造の変化について (关于成本结构的变化):**
「動画プレイヤーなどの『画面を作る費用』は確かに削減できます。しかし、代わりに**『異なるシステム同士を安全に繋ぐ費用』**が新たに発生します。」
(确实可以削减视频播放器等“制作界面”的费用。但是，取而代之的是新产生了**“安全连接不同系统”的费用**。)

**リスクとUXについて (关于风险与用户体验):**
「特に重要なのが**『シングルサインオン（自動ログイン）』**です。これがないと、ユーザーは毎回2回ログインする必要があり、利便性が大きく損なわれます。また、受講履歴が正しく反映されないと、資格認定に支障が出ます。」
(特别重要的是**“单点登录（自动登录）”**。如果没有这个，用户每次都需要登录两次，极大损害便利性。另外，如果学习记录不能正确反映，会影响资格认证。)

**結論の提示 (提出结论):**
「そのため、単純な開発費の削減というよりは、**『開発費を連携費にスライドさせる』**イメージに近くなります。他社システムの利用料も含めたトータルコストで比較検討されることをお勧めします。」
(因此，与其说是单纯削减开发费，不如说是**“将开发费转移到了对接费”**上。建议您结合第三方系统的使用费，进行总成本的比较探讨。)

### 結論 / 结论

| プラン (方案) | 開発費増減 (开发费增减) | 評価 (评价) |
| :--- | :--- | :--- |
| **C-1 (自社主導)** | **+48万円** | **推奨**。コスト増を最小限に抑えつつ、システム健全性を維持できる。 |
| **C-2 (他社主導)** | **+90万円** | **非推奨**。開発費が大幅に上がり、将来のメンテナンス負荷も高い。 |

※いずれの場合も、他社システムの**月額利用料**が別途発生するため、トータルコストはプランA/Bよりも高くなる可能性が高いです。

无论哪种情况，由于还需额外支付第三方系统的**月度使用费**，总成本大概率高于自主开发方案（方案A/B）。
